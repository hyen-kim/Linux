04 리눅스 명령 사용

2주차 - 리눅스의 파일과 디렉터리에 관련된 기본 명령어를 학습합니다.

학습목표:
  •리눅스 파일의  종류와 특징을 이해한다.
  •디렉터리(폴더)  관련 명령  사용법을 익힌다.
  •파일  관련 명령  사용법을  익힌다.

* 반드시 본인의 수업 시간에 맞춰 "출석합니다" 댓글 달아주세요.

  디렉터리를 만들고 작업하세요.

< 프롬프트 기호와 홈 디렉터리 >
프롬프트 
- 사용자의 명령 입력을 기다리는 표시
- 셸에 따라 다르게 나타남 : 배시 셸의 경우 $로 표시, 
                                      시스템 관리자의 경우 #으로 표시

기본디렉터리
- 터미널 창의 기본 디렉터리는 사용자의 홈 디렉터리
- user1@localhost~:user1은 사용자 계정 이름, localhost는 호스트 이름, ~는 user1 사용자의 홈 디렉터리 
- 사용자의 홈 디렉터리는 사용자 계정을 등록할 때 지정

< 명령행 편집 방법 >
- 프롬프트 다음에 명령을 입력하여 리눅스 사용
- 문자 지우기 : 백스페이스키나 delete키를 사용
- 단어 지우기 : ctrl + w, 단어는 공백문자로 구분
- 행 지우기 : ctrl + u

< 명령의 구조 >
- 형식 : 명령 [옵션] [인자]
- 명령 : 리눅스를 사용하기 위해 사용자가 입력하는 다양한 명령
	date(시간과 날짜를 보여줌), man(도움말기능), *ls(디렉토리안의 파일의 리스트를 보여줌), 
	cp, mv등 수백가지가 있다.

< 명령 사용법 >
- 명령만 사용할 경우 : ls
- 명령과 옵션을 사용한 경우 : ls -a => 숨겨진 파일까지 표시
			.파일명 => 숨겨진 파일을 말함
			. => 자기자신
			.. => 상위 ( 부모 )
- 명령과 인자를 사용한 경우 : ls /tmp => tmp라는 디렉토리 안의 내용을 표시
- 명령과 옵션과 인자를 사용한 경우 : ls -a /tmp => tmp 디렉토리의 모든 내용 표시

< 기초 명령 사용법 >
- date 명령 : 현재 날짜와 시간을 출력
- clear 명령 : 현재 보이는 화면을 지우고 커서를 화면 좌측 상단으로 이동

- man 명령 : 각종 명령의 사용법을 알려줌
	=> man 명령 ( 인자 )
	ex) man clear : clear의 자세한 사용법을 볼 수 있음
- passwd 명령 : 사용자 계정의 비밀번호 변경
	=> passwd [인자]
	ex) passwd user 1 

< 터미널 종료하기 > => 두가지 방법이 존재
- exit
- ctrl + d


< 리눅스 파일의 종류와 특징 >

파일의 종류
- 파일 : 관련 있는 정보들의 집합
- 리눅스 파일을 효과적으로 관리하기 위해 디렉터리를 사용
- 파일 시스템 : 디렉터리와 파일로 구성된 전체 집합
- 리눅스에서 파일은 사용 목적에 따라 일반 파일, 디렉터리, 심벌릭 링크, 정치 파일로 구분

1. 일반파일 
- 데이터를 저장하는데 주로 사용
- 각종 텍스트 파일, 실행파일, 이미지 파일 등 리눅스에서 사용하는 대부분의 파일은 일반파일에 해당
- 실행 파일이나 이미지 파일의 경우 바이너리 형태로 데이터가 저장되어 바이너리 파일이라고도 함

2. 디렉터리
- 리눅스에서는 디렉터리도 파일로 취급
- 디렉터리 파일에는 해당 디렉터리에 저장된 파일이나 하위 디렉터리에 대한 정보가 저장

3. 심벌릭링크
- 원본 파일을 대신하여 다른이름으로 파일명을 지정한 것 ( 윈도의 바로가기와 비슷함)

4. 장치파일
- 하드디스크나 키보드 같은 각종 장치도 파일로 취급
- 리눅스 시스템에 부착된 장치를 관리하기 위한 특수 파일

< 작업 디렉터리 >
- 현재 사용중인 디렉터리를 작업 디렉터리 또는 현재디렉터리 라고 함
- 현재 디렉터리 '.' 기호로 표시
- 현재 디렉터리 위치 : pwd 명령으로 확인

< 홈 디렉터리 >
- 처음 사용자 계정을 만들 때 지정
- 파일이나 서브 디렉터리를 생성하여 작업 가능
- 홈 디렉터리는 '~' 기호로 표시

< 절대 경로명과 상대 경로명 >
경로명 : 파일 시스템에서 디렉터리 계층 구조에 있는 특정 파일이나 디렉터리의 위치 표시
경로명에서 각 경로를 구분하는 구분자로 슬래시(/) 사용
경로명에서 가장 앞에 있는 /는 루트 디렉터리를 뜻하지만 경로명 중간에 있는 /는 구분자이다.

1. 절대 경로명 : 항상 루트(/) 디렉터리부터 시작
- 반드시 /으로 시작
- 특정위치를 가리키는 절대경로는 항상 동일

2. 상대 경로명 : 현재 디렉터리를 기준으로 시작
- / 이외의 문자로 시작
- 서브 디렉터리로 내려가려면 서브 디렉터리명을 추가
  상위 디렉터리로 가려면 .. 추가
- 현재 디렉터리가 어디냐에 따라 달라짐

< 현재 디렉터리가 user1 일 때 >
user1이 절대 경로 : /home/user1 
다운로드의 절대 경로 : /home/user1/다운로드
다운로드의 상대 경로 : 다운로드 또는 ./다운로드
hosts 파일의 상대 경로명 : ../../etc/hosts
*) 상위 디렉터리로 갈때 .. 두개 꼭 붙이고 다른 디렉터리로 이동할 땐 /


< 파일과 디렉터리 이름의 규칙 >
- / 를 사용할 수 없음
- 알파벳, 숫자, 붙임표, 밑줄, 점만 허용
- 공백문자를 포함해서 *, |, ",", @ 등등 사용하면 안됨
- 대소문자를 구분 
- 마침표를 시작하면 숨김 파일로 간주	=> ls -a로 확인해보면 됨

< 파일과 관련된 명령 >
- 파일의 내용을 보는 명령 : cat
- 파일을 복사하는 명령 : cp
- 파일을 삭제하고 이동하는 명령 : rm, mv
- 하드 링크와 심벌릭 링크를 생성하는 명령 : ln
- 빈 파일을 만드는 명령 : touch
